Зачем эта страница

Используйте эту страницу как вводный обзор и песочницу для экспериментов с заголовками Host и X-Forwarded-Host. Ниже — базовый пример HTTP‑запроса и интерактивная зона, где можно поиграть с маршрутом одного TLS‑соединения.

Базовый HTTP‑запрос

GET /intranet HTTP/1.1
Host: example.com
X-Forwarded-Host: intranet.example
single TLS channel

Один TLS‑канал, но разные значения Host на разных этапах могут приводить к неожиданной маршрутизации и открытию внутренних путей.

Соберите свой запрос

Внешний балансировщик часто доверяет только полю Host в первом запросе.

Часто используется для прокидывания исходного хоста до бэкенда — при неправильной обработке может стать источником бага.

Если первый запрос фиксирует только внешний Host, но далее бэкенд доверяет X-Forwarded-Host, появляются интересные аномалии, в том числе first‑request routing bypass.
