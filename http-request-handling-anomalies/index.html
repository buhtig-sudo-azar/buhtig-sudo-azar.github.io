HTTP Request Handling Anomalies Lab

Учебный лендинг с интерактивными демо по аномалиям обработки HTTP‑запросов, включая обход маршрутизации первого запроса (first‑request routing bypass) и другие нетривиальные кейсы.

Открыть лабораторию

Базовый HTTP‑запрос

GET /intranet HTTP/1.1
Host: example.com / intranet.example
X-Forwarded-Host: intranet.example
single TLS channel

Этот пример иллюстрирует ситуацию, когда один и тот же TLS‑канал используется для разных значений Host, что может приводить к неожиданным маршрутам и уязвимостям.

Для чего нужна лаборатория

Демки помогают разобрать аномалии маршрутизации, заголовков и балансировщиков. Используйте лабораторию как песочницу для исследования поведения прокси, балансировщиков и бэкендов при нестандартных HTTP‑последовательностях.

Пример цепочки запросов

curl -k https://example.com/intranet

GET /intranet HTTP/1.1
Host: example.com
X-Forwarded-Host: intranet.example
X-Original-Host: example.com

# first request: external host
# routing bypass: internal host

Логика прокси и балансировщика может зависеть от первого запроса в соединении, а последующие запросы с другими заголовками Host приводят к обходу ожидаемой маршрутизации.

Клиент
браузер

Прокси / балансировщик
проверка только первого запроса,
пример: example.com

Бэкенд
внутренний хост: intranet.example

Одно TLS‑соединение, но разные значения Host могут приводить к разным маршрутам и различным последствиям для безопасности.
